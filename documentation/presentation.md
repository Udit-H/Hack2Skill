## 1. Brief About the Idea

### What is Sahayak?

Sahayak (Hindi for "helper") is an AI-powered legal crisis navigator that helps individuals facing housing emergencies вҖ” illegal eviction, domestic violence, and elder abuse вҖ” navigate complex legal procedures, generate official court-ready documents, and find emergency shelter. All through a simple chat interface, in their own language, with trauma-informed design at its core.

In India, over 70% of legal cases involve people who cannot afford lawyers. When someone receives an eviction notice, they face a cascade of crises: they need to understand their legal rights, file the correct police complaints, petition courts, find emergency shelter, and do all of this under extreme time pressure and emotional distress. Sahayak automates this entire journey end-to-end.

### Why AI is Required in This Solution

1. **Complex Legal Reasoning Under Pressure**: A person facing eviction needs to understand which sections of the Bharatiya Nyaya Sanhita (BNS) apply, what documents to file, in what order, at which court вҖ” and they need this within hours, not weeks. Traditional search or static guides fail here because every situation has nuances (DV vs eviction vs senior citizen abuse). AI provides real-time, context-aware legal reasoning that adapts to each user's unique situation.

2. **Multi-Agent Coordination**: No single prompt or workflow can handle the full crisis lifecycle. Sahayak uses 4 specialized AI agents (Triage, Shelter, Legal, Drafting) coordinated by an orchestrator. The Triage Agent classifies the crisis and extracts structured data. The Legal Agent conducts a multi-turn interview to gather case-specific details. The Drafting Agent generates actual court petitions. This agent pipeline is only possible with AI вҖ” each agent uses LLM reasoning to decide what to ask next, when to hand off, and what documents to generate.

3. **Natural Language Interface for Legal Complexity**: Legal procedures are written in dense legalese. Sahayak translates this into a conversational chat вҖ” users describe their situation in plain language (even in Hindi or Kannada), and the AI extracts structured legal data, identifies the correct procedures, and generates documents. No forms to fill, no legal jargon to understand.

4. **Document Intelligence**: Users can upload eviction notices, rental agreements, or court orders. AWS Textract extracts text via OCR, and the AI classifies the document, extracts key dates and amounts, and uses this information to auto-populate legal petitions вҖ” something impossible without AI-powered document understanding.

5. **RAG-Powered Legal Knowledge**: Indian law changes frequently. Sahayak uses a Retrieval-Augmented Generation (RAG) pipeline with 20+ curated legal knowledge documents covering BNS sections, Karnataka Rent Act, Protection of Women from Domestic Violence Act, and Maintenance and Welfare of Parents and Senior Citizens Act. The AI doesn't hallucinate legal advice вҖ” it retrieves from verified sources and grounds its responses in actual law.

### How AWS Services Are Used Within the Architecture

| AWS Service | Role in Sahayak | Why This Service |
|---|---|---|
| **Amazon Bedrock** | Primary LLM вҖ” Meta Llama 3.2 90B Instruct via Converse API | Powers all 4 agents' reasoning, structured data extraction, and legal analysis. Bedrock provides managed inference with no infrastructure to maintain. |
| **Amazon DynamoDB** | Persistent storage вҖ” Chat messages table + Shelters table with geohash GSIs | Stores conversation history for session recovery, shelter data with geospatial indexing (geohash4/5/6) for proximity-based queries. Serverless, scales to zero. |
| **Amazon S3** | PDF document storage вҖ” Generated legal drafts with presigned download URLs | Stores court petitions and legal documents generated by the Drafting Agent. Presigned URLs provide secure, time-limited access without authentication. |
| **Amazon Textract** | OCR вҖ” Extracts text from uploaded eviction notices, rent agreements, court orders | Converts uploaded images/PDFs into structured text that feeds into the legal analysis pipeline. Supports async processing for multi-page PDFs. |
| **Amazon Cognito** | User authentication вҖ” Sign up, login, session management | Provides secure authentication for the React frontend with JWT tokens. Manages user identity for session ownership verification. |
| **AWS App Runner** | Backend deployment вҖ” Hosts the FastAPI server with auto-scaling | Runs the Python backend with automatic scaling, health checks, and zero-downtime deployments from GitHub. |
| **AWS Amplify** | Frontend deployment вҖ” Hosts the React SPA with CDN | CI/CD pipeline for the React frontend with global CDN distribution, custom domains, and environment variable management. |

### What Value the AI Layer Adds to the User Experience

1. **From Weeks to Minutes**: What would normally require a lawyer consultation (вӮ№2,000-5,000), multiple court visits, and weeks of paperwork is compressed into a 10-minute chat conversation. The AI generates court-ready documents in real-time.

2. **Empathy-Driven Interaction**: The AI is specifically prompted with trauma-informed design principles. It never uses judgmental language, provides content warnings before sensitive topics, limits choices to prevent decision paralysis, and maintains a supportive tone throughout.

3. **Intelligent Crisis Detection**: The Triage Agent doesn't just classify вҖ” it detects cascading risks. An eviction case may reveal domestic violence indicators, triggering both legal protection workflows AND emergency shelter search simultaneously. No static system can do this.

4. **Multi-Language Access**: Users can chat in English, Hindi, or Kannada. The AI understands and responds in the user's preferred language, making legal help accessible to non-English speakers who are disproportionately affected by these crises.

5. **Progress Visibility**: Users see real-time progress as the Drafting Agent generates each document вҖ” "Generating Police Complaint (1/3)... Generating Legal Aid Application (2/3)..." вҖ” reducing anxiety during the waiting period.

6. **One-Click Safety Exit (Panic Button)**: For domestic violence survivors, a panic button instantly wipes all session data and redirects to Google вҖ” critical for users whose abusers may check their phone.

### Cost per User Session

At 1,000 sessions/month: **~$0.03-0.05 per complete session** (triage вҶ’ shelter вҶ’ legal вҶ’ drafting)

This is compared to the average legal consultation cost of вӮ№2,000-5,000 ($25-60) in India.

### Built-In Cost Optimization Strategies

| Strategy | Mechanism | Estimated Savings |
|---|---|---|
| **RAG Retrieval Cache** | DynamoDB cache of embedвҶ’searchвҶ’rerank pipeline results (SHA-256 query hash, 24h TTL). Repeated legal queries skip Cohere + ChromaDB entirely. | ~40-60% Cohere API cost reduction |
| **L2 Memory Summarization** | Older conversation turns are summarized into a compact episodic summary (DynamoDB). Only the summary + last 8 messages are sent to Bedrock, instead of the full 20+ turn history. | ~30-50% Bedrock input token savings on long sessions |
| **In-Session RAG Caching** | Legal Agent caches RAG retrieval result in agent state (`current_state.retrieved_legal_context`). Second call to RAG within the same session is free. | 50% RAG calls eliminated per session |
| **Automatic LLM Failover** | Bedrock (primary) вҶ’ Groq (fallback). Groq free tier absorbs overflow traffic during Bedrock throttling/outages. | Prevents failed requests; free tier offloads burst traffic |
| **DynamoDB On-Demand** | Pay-per-request billing вҖ” zero cost when idle, scales automatically during spikes. | No idle capacity cost |
| **Browser-Native TTS/STT** | Uses Web Speech API for voice input/output instead of AWS Polly/Transcribe. | $0 for voice features |
